# 我的与目标设置模块

## 模块目标

管理用户基础信息、饮食目标、活动水平和每日摄入目标。

## 当前问题

当前已有个人目标和基础代谢相关设置。后续如果加入账号或云同步，需要明确哪些数据只保存在本地，哪些需要同步。

## 本轮允许修改

- 我的页面。
- 目标设置。
- 活动水平设置。
- 每日目标计算。
- 与 localStorage 相关的用户配置。

## 本轮禁止修改

- 不改菜谱数据。
- 不改拍照识别。
- 不接账号系统，除非任务明确要求。
- 不引入云数据库，除非任务明确要求。

## 依赖模块

- 数据储存模块。
- 今日记录模块。

## 验收标准

- 用户目标可修改并保存。
- 页面刷新后目标保留。
- 今日记录目标值随设置更新。
- 移动端显示正常。

## 预览检查点

- 我的页面。
- 目标设置控件。
- 今日记录目标值。

## 后续计划

未来如引入 Supabase，可把个人目标和历史记录迁移到云端同步。
