# 食材与库存模块

## 模块目标

管理 Light Flow 的食材基础数据、用户已有食材、食材分类、单位、图标和未来库存能力。

## 当前问题

当前顶部食材列表和菜谱引用的食材 ID 不完全一致。部分菜谱使用了未出现在顶部食材列表中的 ID。

详见：

```text
docs/10-current-state-audit.md
```

## 本轮允许修改

- 食材列表。
- 食材分类。
- 食材单位。
- 食材图标引用。
- 用户已选食材和库存相关逻辑。

## 本轮禁止修改

- 不扩充菜谱，除非任务明确要求。
- 不改拍照识别。
- 不接 Grocy，除非任务明确要求。
- 不删除已有图标资产。

## 依赖模块

- 菜谱数据扩充模块。
- 菜谱图片匹配模块。
- Grocy 集成模块。

## 验收标准

- 所有菜谱引用的食材 ID 都存在。
- 每个食材有中文名、分类和单位。
- 食材图标能正常加载。
- 食材页筛选和选择正常。

## 预览检查点

- 食材页。
- 首页可做菜谱数量。
- 发现页匹配结果。
- 菜谱详情食材用量。

## 后续计划

第一阶段先统一本地食材数据。未来再考虑从 Grocy 同步库存和保质期。
