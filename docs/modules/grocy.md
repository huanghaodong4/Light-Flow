# Grocy 集成模块

## 模块目标

评估并实现 Grocy 作为外部家庭库存来源，让 Light Flow 可以读取库存、保质期和购物清单相关数据。

## 当前问题

Grocy 还未接入。当前只是方向讨论，详见：

```text
docs/06-grocy-integration-notes.md
```

## 本轮允许修改

- Grocy 集成方案文档。
- 食材 ID 映射设计。
- 库存同步接口设计。
- 未来 API 同步层。

## 本轮禁止修改

- 不直接复制 Grocy 代码。
- 不复制 Grocy UI。
- 不让 Light Flow 变成重型库存 ERP。
- 不用 Grocy 管菜谱图片。

## 依赖模块

- 食材与库存模块。
- 首页推荐模块。
- 菜谱模块。

## 验收标准

- Grocy 商品能映射到 Light Flow 食材 ID。
- 库存数据能影响可做菜谱。
- 临期食材可以提高推荐优先级。
- 缺少食材可以进入购物清单方案。

## 预览检查点

- 食材页库存状态。
- 首页推荐排序。
- 发现页可做菜谱。
- 缺少食材提示。

## 后续计划

Grocy 不是第一阶段优先级。应在菜谱图片、菜谱扩充和拍照识别基础打稳后再做。
