# 样式系统模块

## 模块目标

让 Light Flow 在页面增加和功能迭代后仍保持统一、轻量、移动端优先的视觉体验。

## 当前问题

当前样式集中在 `styles.css`。后续如果页面和状态继续增加，需要更清晰的样式边界。

## 本轮允许修改

- 局部页面样式。
- 设计变量。
- 组件复用样式。
- 与模块相关的响应式修复。

## 本轮禁止修改

- 不重新设计全站，除非任务明确要求。
- 不引入大型 UI 组件库。
- 不让新页面变成后台系统风格。
- 不做无关视觉重写。

## 依赖模块

- 首页推荐模块。
- 菜谱模块。
- 拍照识别模块。
- 今日记录模块。

## 验收标准

- 移动端文字不溢出。
- 卡片不互相挤压。
- 底部导航不遮挡关键内容。
- 新增功能符合 Light Flow 当前视觉语言。

## 预览检查点

- 390px 移动端视口。
- 首页。
- 发现页。
- 拍照页。
- 记录页。
- 菜谱详情弹窗。

## 后续计划

React/Vite 迁移后，可逐步引入 CSS Variables 或 CSS Modules，避免全站样式继续膨胀。
