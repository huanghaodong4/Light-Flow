# 菜谱图片匹配模块

## 模块目标

保证菜谱名称、主食材、菜品形态和图片一致，恢复用户对菜谱内容的信任。

## 当前问题

详见：

```text
docs/11-recipe-image-audit.md
```

当前存在若干明显图文不一致菜谱，例如豆腐蔬菜煲使用麻婆豆腐图、牛肉糙米饭使用牛肉面图等。

## 本轮允许修改

- `app.js` 中的菜谱图片路径。
- 必要时调整菜名、描述、required 食材，让它与图片一致。
- 本模块相关审计文档。

## 本轮禁止修改

- 不改拍照识别流程。
- 不扩充新菜谱，除非任务明确要求。
- 不重做全站样式。
- 不删除图库原图。

## 依赖模块

- 菜谱数据扩充模块。
- 食材与库存模块。

## 验收标准

- 所有“不合格”图片都有处理方案。
- 修复后的菜谱图文没有明显错配。
- 图片路径能正常加载。
- `node --check app.js` 通过。

## 预览检查点

- 首页推荐卡片。
- 发现页菜谱卡片。
- 菜谱详情弹窗大图。

## 后续计划

修复现有错配后，再基于图库反推新增菜谱，避免继续制造图文不一致。
