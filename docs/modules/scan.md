# 拍照识别模块

## 模块目标

把拍照识别从占位入口升级为可上传、可预览、可估算、可校正、可写入记录的流程。

## 当前问题

当前拍照页只有上传入口和说明，没有真实识别结果和记录闭环。

详细方案见：

```text
docs/07-photo-food-recognition-and-calorie-plan.md
```

## 本轮允许修改

- 拍照页 UI 和状态。
- 模拟识别结果。
- 识别结果编辑和确认逻辑。
- 与今日记录的写入连接。
- 拍照页局部样式。

## 本轮禁止修改

- 不改菜谱数据。
- 不改发现页。
- 不接真实 AI API，除非任务明确要求。
- 不把 API Key 放前端。

## 依赖模块

- 今日记录模块。
- 数据储存模块。
- AI 后端模块。

## 验收标准

- 可以上传或选择照片。
- 可以显示照片预览。
- 可以显示 1-5 个模拟识别项。
- 可以修改份量。
- 可以删除错误项。
- 可以新增漏识别食物。
- 可以写入今日记录。

## 预览检查点

- 空状态。
- 上传预览状态。
- 识别中状态。
- 待确认状态。
- 已记录状态。
