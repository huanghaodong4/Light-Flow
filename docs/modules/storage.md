# 数据储存模块

## 模块目标

明确 Light Flow 的数据分层，避免用户数据、静态数据、AI 数据和外部库存数据混在一起。

## 当前问题

当前项目主要依赖 `app.js` 和浏览器本地状态。后续菜谱扩充、拍照识别和云同步会让数据边界更复杂。

## 本轮允许修改

- localStorage 读写封装。
- 用户状态结构。
- 数据迁移兼容逻辑。
- 存储相关文档。

## 本轮禁止修改

- 不引入数据库，除非任务明确要求。
- 不把大图存进 localStorage。
- 不存 API Key。
- 不重构无关页面。

## 依赖模块

- 拍照识别模块。
- 今日记录模块。
- 食材与库存模块。

## 验收标准

- 用户数据能正常读取和保存。
- 页面刷新后状态保留。
- 不存储大体积原始图片。
- 不暴露敏感配置。

## 预览检查点

- 食材选择。
- 收藏菜谱。
- 今日记录。
- 个人目标。

## 后续计划

第一阶段继续使用 localStorage。未来如需账号和多设备同步，再评估 Supabase。
